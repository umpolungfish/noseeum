## Project Overview

Noseeum is a security framework for Unicode-based encoding and obfuscation techniques. It provides both red capabilities (encoding/obfuscation) and blue capabilities (detection/scanning). The framework uses advanced Unicode smuggling techniques including Trojan Source (Bidi), homoglyph substitution, invisible character encoding, and language-specific exploits.

## Development Environment Setup

### Virtual Environment
Always activate the virtual environment before running commands:
```bash
source venv/bin/activate
```

### Initial Setup
Required before first use or after clean install:
```bash
# Generate required data files (must be done before installation)
python3 create_registry.py      # Creates homoglyph_registry.json
python3 create_nfkc_map.py      # Creates nfkc_map.json

# Install the package
make install                     # Or: pip install .
```

### Clean and Reinstall
```bash
make clean                       # Remove build artifacts
make uninstall                   # Remove installed package
make install                     # Reinstall
```

## Architecture

### Modular Design
The framework uses a plugin-based architecture with three main layers:

1. **Core Engine Layer** (`noseeum/core/`)
   - `engine.py`: Central ObfuscationEngine class that orchestrates all obfuscation modules
   - `grammar_db.py`: Language-specific grammar and vulnerability database
   - `stealth_engine.py`: Evasion and anti-detection techniques
   - `evasion_library.py`: Collection of evasion tactics
   - `integration.py`: Module integration and coordination

2. **Attack Modules Layer** (`noseeum/attacks/`)
   - **Basic attacks**: bidi.py, homoglyph.py, invisible.py, language.py
   - **Advanced attacks**: normalization.py, unassigned_planes.py, payload_injection.py, hangul_encoding.py
   - **Language-specific**: go_attack.py, kotlin_attack.py, javascript_attack.py, swift_attack.py

3. **Detection Layer** (`noseeum/detector/`)
   - `scanner.py`: File scanning for Unicode vulnerabilities

### CLI Structure
The CLI is organized hierarchically using Click command groups:

```
noseeum
├── detect              # Scan files for vulnerabilities
├── info                # Show framework information
├── techniques          # List obfuscation techniques
├── vulnerabilities     # Show language-specific vulnerabilities
└── attack              # Attack command group
    ├── bidi            # Trojan Source (Bidi) attack
    ├── homoglyph       # Homoglyph substitution
    ├── invisible       # Invisible character encoding
    ├── language        # Language-specific basic attacks
    ├── advanced        # Advanced techniques subgroup
    │   ├── normalization
    │   ├── unassigned-planes
    │   ├── payload-injection
    │   └── hangul-encoding
    └── language-specific   # Language-specific attacks subgroup
        ├── go-attack
        ├── kotlin-attack
        ├── javascript-attack
        └── swift-attack
```

### Key Design Patterns

**ObfuscationModule Abstract Base Class**: All attack modules inherit from `ObfuscationModule` (defined in `noseeum/core/engine.py`) which requires:
- `get_name()`: Return technique name
- `get_description()`: Return technique description
- `get_supported_languages()`: Return list of LanguageSupport enums
- `obfuscate(content, target_language, **kwargs)`: Execute the obfuscation

**Enumerations**:
- `ObfuscationTechnique`: Defines all supported techniques (BIDI, HOMOGLYPH, INVISIBLE, etc.)
- `LanguageSupport`: Defines supported languages (PYTHON, JAVASCRIPT, JAVA, GO, KOTLIN, SWIFT, etc.)

## Data Files

### Required Generated Files
These files MUST be generated before using the framework:
- `homoglyph_registry.json`: Maps characters to visually similar Unicode homoglyphs (generated by `create_registry.py`)
- `nfkc_map.json`: NFKC normalization mappings (generated by `create_nfkc_map.py`)

### Configuration Files
- `config.json`: Framework configuration including detector settings and confusable scripts
- `noseeum/data/NOSEEUMS_list.json`: Registry of suspicious Unicode characters
- `noseeum/data/homoglyph_registry.json`: Package copy of homoglyph registry
- `noseeum/data/nfkc_map.json`: Package copy of NFKC map

## Common Commands

### Detection/Scanning
```bash
# Scan a file for Unicode vulnerabilities
noseeum detect --file /path/to/file.js

# The scanner detects:
# - Bidirectional control characters (U+202A-202E, U+2066-2069)
# - Zero-width characters (U+200B-200D, U+FEFF, U+00AD)
# - Mixed-script homoglyphs in identifiers
```

### Framework Information
```bash
# Show registered modules and supported languages
noseeum info

# List all obfuscation techniques with descriptions
noseeum techniques

# Show vulnerabilities for a specific language
noseeum vulnerabilities --target-language python
```

### Attack Execution
Attack commands vary by module. Common pattern:
```bash
noseeum attack <module> [OPTIONS]
```

Refer to `docs/USAGE.md` for complete command documentation.

## Important Implementation Notes

### File Encoding
The `bidi.py` module includes `read_file_with_encoding()` which tries multiple encodings (utf-8, latin-1, cp1252, ascii) when reading target files. Use this pattern when dealing with files of unknown encoding.

### Payload Sanitization
Attack modules sanitize payloads to prevent breaking injections. The `sanitize_payload()` function in `bidi.py` demonstrates this pattern:
- Remove/replace newlines, carriage returns, tabs
- Escape quotes that could break string literals
- Language-specific sanitization as needed

### Error Handling
Use utilities from `noseeum/utils/error_handlers.py`:
- `validate_line_number()`: Validate line number is within file bounds
- `handle_file_error()`: Standardized file error handling

### Configuration Access
Use `noseeum.config.get_config(key, default)` to access configuration values from `config.json`.

## Adding New Attack Modules

1. Create module file in `noseeum/attacks/`
2. Implement `ObfuscationModule` abstract base class from `noseeum/core/engine.py`
3. Define a Click command decorated with `@click.command()`
4. Import in `noseeum/cli.py`
5. Register command with appropriate command group (attack/advanced/language_specific)
6. Update `ObfuscationTechnique` enum in `noseeum/core/engine.py` if adding new technique type
